# Context

`@agent` in the chat composer is only for digital employees in this phase. It
must not switch the renderer to `agent:{agentId}:main`, must not call
`sessions_spawn`, and must not create a child session.

# Required Behavior

- Renderer keeps the current `sessionKey` and sends one normal `chat.send`.
- `chat.send` carries `executeAsAgentId` for the target digital employee.
- The Gateway streams all delta/tool/final events under the current `sessionKey`.
- The runtime uses the current session model.
- The runtime uses target digital employee workspace:
  `~/.openclaw/workspace-{agentId}`.
- Prompt, workflow, Skill, and MCP resources come only from
  `digital-employee/{agentId}/` or the installed digital employee directory.
- No global Skill/MCP fallback is allowed for digital employee execution.
- Assistant messages generated for this run are tagged with `executedByAgentId`;
  UI labels prefer the digital employee display name and fall back to `agentId`.

# Validation

- Unit tests cover renderer send params: `chat.send`, current session key,
  `executeAsAgentId`, no `tools.invoke/sessions_spawn`.
- Unit tests cover current-session streaming messages being annotated with
  `executedByAgentId`.
- Harness validation should pass for this task spec.
